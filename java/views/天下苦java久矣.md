这个问题得放到时代背景去看：因为电商崛起的年代，cpp 虽然牛逼但开发效率太低了赶不上市场节奏，php 遍地且开发效率贼高但奈何性能不行，所以你看到为啥早期阿里用 php 然后转 java 。
等到阿里用 java 大大优化了性能并且行业一哥，业务量已经庞大到无法随便切换技术栈了。
而 bat 作为国内一个时代的技术标签，阿里系已经众多 java 跟随者，就把 java 社区越做越大了，加之 java 亲爹搞那些轮子确实社区友好，所以 java 作为性能、开发效率 balance 的佼佼者，大量占领了市场。
所以，php 性能要是罩得住，八成没 java 什么事； cpp 开发效率要是够给力，八成没 java 什么事；其他性能和开发效率兼具的语言比如 nodejs 、golang 要是能早点火并且社区轮子逐渐跟上，java 也不至于一家独大了十几年。

可喜的是，一波新语言在崛起，尤其是 golang 、既能云原生、也能 curd ，性能很高、消耗却低（不至于像内存杀手 java 那样大点的业务量动辄就 OOM 警告）
那些天天喷 golang 的人连 golang 都没玩明白就说开发效率低，我们一票日行一千夜写八百、简单 curd 业务能让键盘冒火星子的高效 gopher 都懒得喷这些 golang 小白。说他们 golang 小白并不是说他们其他语言水平差，只是沉没成本效应、很多这种“大聪明”一旦舒适区里呆久了看到其他不同于自己习惯的新事物就很难跳出舒适圈很难切换过去了，其实这也叫聪明反被聪明误、你会很多所以你尾大不掉所以你耽误了自己的业务能力、尤其是性能和消耗。你天天抱着 java 搞来搞去也优化不了什么并发量、降低不了内存消耗、只能靠堆机器，更有甚者，还造出那种“堆机器就能解决”的词用来洋洋得意，说得好像那是他们多高的本事似的。然而机器数量是别人几倍、不花企业成本吗、不耗电给环保增加压力吗？

java 的市场定位比 cpp 、php 好，在那个年代既是很多其他语言没有成熟、崛起的档口，又是电商等企业级崛起的档口，开发效率、性能 balance 非 java 莫属，可谓天时、地利、人合（其他语言不给力相当于反向帮助 java ）占全了

Sun 爸爸当年对技术语言的市场定位也是非常清醒，市场思维远超 cpp 委员会那帮老家伙整天搞学术搞茴字的六种写法，所以也是活该 java 霸榜那么多年。要不是技术大跃进、消费者业务崛起后移动端、ai 、前端纷纷爆炸式增长、内核离不开 c ，ai 离不开 py 、前端离不开 js ，后续 java 还得继续霸榜。当然，云时代也让 golang 搅一把、虽然目前整体没抢到太多份额但确实开辟了更多赛道。

不过话说回来，java 毕竟借东风跟随电商业务经受了万亿级的流量考验，所以要说他们天天八股搞这个倒也算是实际生产需要，但本质上，java 这些堆机器解决的只是无状态接口类为主的业务，而真正扛鼎的确实背后那些基础设施，数据库、缓存、搜索，还有很多是工程层面的比如运维层面，而架构设计（主要是指软件分布式服务的架构）本身对于接口这种服务，各种语言其实都差不多可以实现、所以应该是语言无关的。

相比于其他语言，八股背得最多，所以嘴炮最强。但我见过很多八股强者、自己坦诚：背会了很多，工程里却不知道怎么用。
当然，也有强者懂得从实际出发解决工程难题，只是比例太小了些，而且多数也是旁通系统、c/cpp 那些，并不是只搞 java 。
javaer 真正的强，是强于社区爸爸提供大而全的轮子。

鹅系早期 IM 也好，后来游戏发力也好，这些都是 cpp 技术栈偏重些，给我的感觉人家就没像电商系那样作为 java 的带头大哥把八股社区搞得乌烟瘴气口嗨无敌。
